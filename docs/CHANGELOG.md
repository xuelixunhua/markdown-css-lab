# Changelog

## Unreleased

### Added

- 建立本地微信公众号 Markdown 渲染器：左侧 Markdown / CSS 编辑，右侧公众号宽度预览。
- 接入三套 CSS 主题：公众号、PDF 手册、技术文档。
- 支持一键复制富文本到微信公众号后台。
- 支持复制 HTML、导出 HTML。
- 支持 `[TOC]` 自动生成目录。
- 支持将 Markdown 外部链接自动转为正文上标编号，并在文末生成“参考链接”脚注区。
- 支持左侧 Markdown 与右侧预览同步滚动，默认开启，可手动关闭。
- 新增项目级文档：更新日志、功能清单、CSS 认知。

### Changed

- 微信公众号主题的加粗强调色调整为更醒目的橙红色。
- 三套主题的 TOC 目录行距收紧，减少目录高度。
- 复制到公众号时移除固定宽度，避免粘贴后内容区域变窄。

### Fixed

- 修复 Markdown 中“列表标题下一行单独冒号”导致的本地渲染换行问题。
- 修复 Markdown 中“列表标题以冒号结尾，正文在下一行”导致的公众号粘贴换行问题。
- 修复 Markdown 中“列表标题下一行接说明文本”导致的列表项被拆行问题，例如 `- **核心行为**` 后接 `——说明`。
- 增加微信复制兼容处理：复制富文本时将 `strong` / `b` 转成带 inline style 的 `span`。
- 增加列表标题冒号绑定：复制富文本时把列表项开头强调文字后的冒号并入同一个行内节点，降低微信编辑器把冒号拆成新行的概率。
- 增加复制阶段的开头 `<br>` 清理，避免微信保留列表标签后的强制换行。
- 参考 Doocs 的微信兼容思路，在复制阶段将嵌套列表从 `li` 内部移出，减少微信编辑器对嵌套列表结构的二次整理。
- 改为在复制到公众号时将真实 `ul/ol/li` 列表转换为手写编号 / 符号的 `section` 列表，绕开微信公众号对列表项中加粗片段的自动换行。
