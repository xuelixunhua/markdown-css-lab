# CSS Knowledge

## 1. 版心

版心是正文主要阅读区域的宽度和位置。好看的 Markdown 渲染通常不是把内容铺满屏幕，而是让文字稳定落在一个可预测的阅读宽度里。

- PDF 手册适合强版心：页面边距、正文宽度、标题层级都稳定。
- 公众号适合弱版心：外部容器由微信后台决定，CSS 应尽量避免固定宽度。
- 技术文档适合中等版心：正文可读，表格和代码块要有足够空间。

## 2. 字体

衬线字体更有书卷感和纸面感，适合标题、手册、长文的关键层级。无衬线字体更清晰、现代，适合正文、公众号、技术说明。

当前原则：

- 公众号：正文优先无衬线，保证移动端清晰。
- PDF 手册：标题可用衬线，正文保持稳定阅读。
- 技术文档：正文和代码区分明显，代码使用等宽字体。

## 3. 颜色

颜色不是装饰，而是信息层级。

- 正文使用接近黑色但不纯黑的颜色，减轻阅读压力。
- 强调色只用于关键字、链接、标题装饰，不应大面积铺开。
- 背景色用于引用、TOC、代码块时要很淡，避免抢正文。
- 公众号里的加粗需要比普通正文更鲜明，否则用户在手机上看不出重点。

## 4. 正文密度

正文密度由字号、行高、段间距共同决定。

- 公众号：字号 16px 左右，行高 1.65-1.8，段落间距略松。
- PDF 手册：字号可略小，行距更克制，依赖页边距和标题层级形成秩序。
- 研究笔记：密度可以更高，但列表、表格、代码块要清楚分隔。

## 5. 多级标题

标题不是逐级变小那么简单，而是承担不同的导航任务。

- H1：文档标题，出现次数应少。
- H2：大章节，适合明显留白或居中处理。
- H3：正文里的主要小节，适合左侧竖线、加粗或色彩提示。
- H4：段落组标题，适合轻强调。
- H5 / H6：只适合补充层级，尽量低调，避免视觉噪音。

## 6. 微信公众号约束

微信公众号后台会改写或过滤部分 HTML / CSS，所以 CSS 不能只追求浏览器预览好看。

需要特别注意：

- 避免依赖伪元素，因为复制时通常不会进入剪贴板。
- 避免固定宽度，否则粘贴后可能显得很窄。
- 避免复杂布局属性，比如 grid、flex 作为正文核心结构。
- 语义标签如 `strong` 在微信里可能被二次转换，复制时转成带样式的 `span` 更稳定。
- 列表里“强调词 + 冒号 + 正文”是高风险结构，需要在复制时做兼容处理。
- 列表里“强调词 + 冒号 + 换行 + 正文”更高风险，应该在 Markdown 解析前合并成一行。
- 嵌套列表在标准 HTML 中通常写作 `li > ul`，但微信公众号编辑器会二次整理这类结构。Doocs 的做法是复制前把嵌套列表移到 `li` 后面，我们也采用类似兼容策略。
- 当微信仍然把列表项里的加粗标签拆行时，说明问题已经不只是 CSS，而是微信对 `li` 富文本结构的二次编辑。更稳的策略是在复制阶段放弃真实列表语义，改用 `section` + 手写编号 / 项目符号模拟列表。
- 微信公众号正文里的外部链接通常不能直接跳转。更稳的排版原则是：正文保留链接文字和上标编号，文末集中列出“参考链接”，既不破坏阅读节奏，也保留来源。
